# SQUAD Security Policy

## Repository access posture

This repository is **private**. The Auernyx-com GitHub organization is operated by a single owner/operator. No external collaborators have read or write access unless explicitly granted.

## Incident record: baseline system snapshots committed to git

### Date range
December 2025 (snapshot files created) → February 2026 (discovered and remediated)

### What happened
The external `baseline-algorithms-and-programs` tool (invoked via `launchers/SQUAD_BASELINE_PRE.cmd` and `launchers/SQUAD_BASELINE_POST.cmd`) writes Windows system diagnostic snapshots to `OUTPUTS/system_snapshot_<timestamp>/` as part of its session baseline ritual. The `.gitignore` only excluded `OUTPUTS/RUNS/` and did not cover the snapshot directories. Two snapshot sets were committed to git history and included in the initial repo push.

### What data was in the snapshots

| File pattern | Contents |
|---|---|
| `systeminfo.txt` | Hostname (`AESIR101`), registered owner email, Windows Product ID, hardware model (ASUS TUF Gaming F16 FX608JMR), hotfix levels, IP addresses |
| `ipconfig.txt` / `network_ipconfig.txt` | MAC addresses, IPv4/IPv6 addresses, DHCP info, DNS suffix |
| `netstat.txt` / `netstat_ano.txt` | All open TCP connections and listening ports at capture time |
| `processes.txt` | Full running process list with PIDs and CPU times |
| `scheduled_tasks.txt` | All scheduled tasks including Windows user SIDs |
| `system_events*.txt` | Windows event log entries |
| `services_automatic.txt` / `auto_services.txt` | All automatic-start Windows services |
| `admin_clerk_transcript_20251214_214113.log` | PowerShell transcript with Windows username |

Two snapshot sets were affected:
- `OUTPUTS/system_snapshot_20251215_013807/` — 9 files, captured 2025-12-15
- `OUTPUTS/system_snapshot_20251218_054245/` — 9 files, captured 2025-12-18

### Remediation applied (February 2026)
1. All 24 sensitive files removed from git tracking via `git rm --cached`
2. `.gitignore` updated: changed `OUTPUTS/RUNS/` → `OUTPUTS/*` with `!OUTPUTS/README.md` — the entire OUTPUTS directory is now excluded except the README
3. Commit `5a55a5f` on branch `copilot/link-firebase-fork` implements both removals

### Remaining posture
- **Files are still in git history.** The repository is private; only the owner has access to the history.
- **No external exposure occurred** — the repository was private at the time of the initial commit.
- When convenient, history can be permanently purged using `git filter-repo` (see steps below).

### Optional: permanent history purge (run locally when convenient)

Install `git-filter-repo` (pip or package manager), then:

```bash
git filter-repo \
  --path OUTPUTS/system_snapshot_20251215_013807 \
  --path OUTPUTS/system_snapshot_20251218_054245 \
  --path OUTPUTS/admin_clerk_run_20251214_213913.log \
  --path OUTPUTS/admin_clerk_run_20251214_213957.log \
  --path OUTPUTS/admin_clerk_transcript_20251214_214113.log \
  --invert-paths
git push origin --force --all
```

This permanently removes the paths from all commits in history and requires a force-push.

## Ongoing data handling rules

- `OUTPUTS/` is **write-protected** for hand-editing (governance rule). All content under `OUTPUTS/` is generated by governed runs and is **local-only** (excluded from git via `.gitignore`).
- Baseline ritual outputs (system snapshots, process lists, network state) are **never tracked in git**, regardless of how they are named.
- If the baseline tool creates new output directories, verify they fall under the `OUTPUTS/` gitignore before committing.
